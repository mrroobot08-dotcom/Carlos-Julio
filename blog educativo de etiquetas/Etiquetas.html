🏗 Etiquetas Estructurales

Estas etiquetas definen el esqueleto del documento HTML. Le dicen al navegador cómo está organizada la página y qué rol cumple cada bloque.

<html>

Definición

Es la etiqueta raíz que envuelve todo el documento. Indica al navegador que el contenido es HTML. El atributo lang indica el idioma de la página.

Sintaxis básica

<html lang="es">
  <!-- todo el documento va aquí -->
</html>

Uso adecuado

Debe ser la primera etiqueta del documento, después del DOCTYPE. Solo puede haber un <html> por página.

Buenas prácticas

  • Siempre incluir el atributo lang para que los lectores de pantalla pronuncien bien el texto.
  • No olvides el DOCTYPE antes de <html>: <!DOCTYPE html>

<head>

Definición

Contiene información sobre la página que no se muestra visualmente: el título, el charset, enlaces a archivos CSS y descripción para buscadores.

Sintaxis básica

<head>
  <meta charset="UTF-8" />
  <title>Mi página</title>
</head>

Uso adecuado

Va justo después de <html> y antes de <body>. Todo lo que va aquí es para el navegador, no para el usuario.

Buenas prácticas

  • El primer elemento dentro de <head> siempre debe ser <meta charset="UTF-8"> para evitar problemas con tildes y caracteres especiales.
  • Incluir siempre un <title> descriptivo.

<body>

Definición

Contiene todo el contenido visible de la página: textos, imágenes, botones, tablas, formularios y más.

Sintaxis básica

<body>
  <h1>Hola mundo</h1>
  <p>Este es el contenido visible.</p>
</body>

Uso adecuado

Solo puede existir un <body> por documento. Todo lo que el usuario ve en pantalla debe estar dentro del body.

Buenas prácticas

  • Colocar los archivos JavaScript al final del <body> para que la página cargue más rápido.
  • Usar etiquetas semánticas dentro del body (header, main, footer) en lugar de solo divs.

<header>

Definición

Encabezado de la página. Suele contener el logo, el título principal y el menú de navegación.

Sintaxis básica

<header>
  <h1>Nombre del sitio</h1>
  <nav>...</nav>
</header>

Uso adecuado

No confundir con <head>. El <header> es parte del contenido visible de la página; el <head> no.

Buenas prácticas

  • No anidar un <header> dentro de otro <header>.
  • Puede aparecer más de una vez en la página (por ejemplo, dentro de un artículo).

<nav>

Definición

Agrupa los enlaces de navegación principales del sitio. Ayuda a los buscadores y lectores de pantalla a identificar el menú.

Sintaxis básica

<nav>
  <a href="#inicio">Inicio</a>
  <a href="#contacto">Contacto</a>
</nav>

Uso adecuado

Solo para bloques de navegación importantes. No usar para cada grupo de enlaces de la página.

Buenas prácticas

  • Usar listas <ul> dentro del <nav> cuando hay muchos enlaces.
  • El menú debe ser fácil de usar en móviles.

<main>

Definición

Contiene el contenido principal y único de la página. Los lectores de pantalla pueden saltar directamente a él.

Sintaxis básica

<main>
  <section>
    <h2>Contenido principal</h2>
    <p>...</p>
  </section>
</main>

Uso adecuado

Solo debe haber un <main> por página. No incluir el header ni el footer dentro.

Buenas prácticas

  • Siempre usar <main> para el contenido central de la página.
  • Nunca repetirlo; solo uno por documento.

<section>

Definición

Divide el contenido en bloques temáticos. Cada sección agrupa contenido relacionado y suele tener su propio título.

Sintaxis básica

<section id="productos">
  <h2>Nuestros productos</h2>
  <p>Descripción...</p>
</section>

Uso adecuado

Para dividir el contenido de una página en temas. Por ejemplo: sección de servicios, sección de contacto, sección de noticias.

Buenas prácticas

  • Cada <section> debe tener un título (h2, h3...) para identificarla.
  • No usar <section> solo para aplicar estilos; para eso está <div>.

<article>

Definición

Contenido independiente que tiene sentido por sí solo: un post de blog, una noticia, un comentario o una tarjeta de producto.

Sintaxis básica

<article>
  <h2>Título del artículo</h2>
  <p>Contenido del artículo...</p>
</article>

Uso adecuado

Pregunta clave: ¿este contenido tiene sentido si lo sacas de la página? Si la respuesta es sí, usa <article>.

Buenas prácticas

  • Puede tener su propio <header> y <footer> internos.
  • Ideal para blogs, noticias y catálogos de productos.

<footer>

Definición

Pie de página del documento o de una sección. Contiene información de contacto, derechos de autor y enlaces secundarios.

Sintaxis básica

<footer>
  <p>© 2025 Mi sitio web. Todos los derechos reservados.</p>
</footer>

Uso adecuado

Va al final de la página. También puede ir al final de un <article> o <section> para dar información adicional.

Buenas prácticas

  • No colocar contenido principal dentro del footer.
  • Incluir siempre información de contacto o derechos de autor.

<div> y <span>

Definición

<div> es un contenedor de bloque sin significado semántico. <span> es un contenedor en línea, también sin semántica. Se usan para aplicar estilos CSS.

Sintaxis básica

<!-- div: ocupa toda la línea -->
<div class="caja">
  <p>Contenido agrupado</p>
</div>

<!-- span: va dentro del texto -->
<p>El precio es <span class="precio">$50.000</span></p>

Uso adecuado

Usar solo cuando ninguna etiqueta semántica describe bien el contenido. Son el "último recurso" para agrupar elementos.

Buenas prácticas

  • Preferir etiquetas semánticas (section, article, nav...) sobre <div>.
  • Siempre asignar una clase descriptiva: class="tarjeta-producto" en lugar de solo <div>.

✍ Etiquetas de Texto

Estas etiquetas dan jerarquía, formato y significado al contenido escrito de la página.

<h1> hasta <h6>

Definición

Son los encabezados jerárquicos. <h1> es el título más importante de la página; <h6> es el de menor jerarquía.

Sintaxis básica

<h1>Título principal de la página</h1>
<h2>Subtítulo de sección</h2>
<h3>Subtítulo de subsección</h3>

Uso adecuado

Los motores de búsqueda y los lectores de pantalla usan los encabezados para entender la estructura del contenido. No usarlos solo para hacer el texto más grande.

Buenas prácticas

  • Solo un <h1> por página, que sea el título principal.
  • No saltar niveles: si usas h2, el siguiente debe ser h3, no h4.
  • Usa CSS para cambiar el tamaño visual, no los encabezados.

<p>

Definición

Define un párrafo de texto. Es el elemento más común para contenido escrito. El navegador añade espacio arriba y abajo automáticamente.

Sintaxis básica

<p>Este es un párrafo de texto en HTML.
Es un elemento de bloque y ocupa toda la línea.</p>

Uso adecuado

Para cualquier bloque de texto continuo. No usar para encabezados, ítems de listas o celdas de tablas.

Buenas prácticas

  • No usar <br><br> para simular párrafos; usar <p> correctamente.
  • No anidar <p> dentro de otro <p>.

<strong> y <em>

Definición

<strong> resalta texto con importancia fuerte (se ve en negrita). <em> indica énfasis (se ve en cursiva). Ambas tienen significado semántico, no son solo estilo.

Sintaxis básica

<p><strong>Advertencia:</strong> no borres este archivo.</p>

<p>Debes <em>realmente</em> leer las instrucciones.</p>

Uso adecuado

<strong> cuando el texto es importante. <em> cuando el énfasis cambia el significado de la frase. No usarlos solo para hacer negrita o cursiva visual (para eso usa CSS).

Buenas prácticas

  • Preferir <strong> sobre <b> y <em> sobre <i> por su valor semántico.
  • No abusar: si todo está en negrita, nada destaca realmente.

<a>

Definición

Crea un hipervínculo (enlace). Con el atributo href enlaza a otras páginas, archivos, correos o secciones de la misma página.

Sintaxis básica

<!-- Enlace a otra página -->
<a href="https://google.com" target="_blank">Ir a Google</a>

<!-- Enlace a una sección de la misma página -->
<a href="#contacto">Ir a Contacto</a>

<!-- Enlace de correo -->
<a href="mailto:correo@ejemplo.com">Escríbenos</a>

Uso adecuado

Para navegar entre páginas o secciones. target="_blank" abre el enlace en una pestaña nueva.

Buenas prácticas

  • El texto del enlace debe describir el destino, no decir "haz clic aquí".
  • Al usar target="_blank", agregar rel="noopener" por seguridad.

<br> y <blockquote>

Definición

<br> inserta un salto de línea dentro del texto. <blockquote> representa una cita larga tomada de otra fuente.

Sintaxis básica

<!-- Salto de línea -->
<p>Línea uno.<br>Línea dos, mismo párrafo.</p>

<!-- Cita de otra fuente -->
<blockquote cite="https://fuente.com">
  <p>El texto citado de otra fuente va aquí.</p>
</blockquote>

Uso adecuado

<br> para poesía o direcciones postales, no para separar párrafos. <blockquote> solo para citas textuales de otras fuentes.

Buenas prácticas

  • Nunca usar <br><br> para crear espacio; usar CSS con margin o padding.
  • Incluir el atributo cite en blockquote con la URL de la fuente.

<code> y <pre>

Definición

<code> marca fragmentos de código en línea con fuente monoespaciada. <pre> preserva los espacios y saltos de línea del texto.

Sintaxis básica

<!-- Código en línea -->
<p>Usa la propiedad <code>color: red</code> en CSS.</p>

<!-- Bloque de código -->
<pre><code>
  let nombre = "Juan";
  console.log(nombre);
</code></pre>

Uso adecuado

Para mostrar código fuente en documentación, tutoriales o guías técnicas. La combinación <pre><code> es el estándar para bloques de código.

Buenas prácticas

  • Escapar los signos < y > dentro del código: escribir &lt; y &gt;.
  • Aplicar un color de fondo diferente con CSS para que el código sea fácil de identificar.

📋 Etiquetas de Listas

Permiten organizar información en ítems. Hay tres tipos: no ordenadas, ordenadas y de definición.

<ul> — Lista no ordenada

Definición

Lista con viñetas (puntos). Se usa cuando el orden de los elementos no importa.

Sintaxis básica

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Uso adecuado

Para ingredientes, características, menús de navegación o cualquier grupo de ítems donde el orden no importa.

Buenas prácticas

  • Solo los elementos <li> pueden ir directamente dentro de <ul>.
  • Cambiar el tipo de viñeta con CSS: list-style-type: square.

<ol> — Lista ordenada

Definición

Lista numerada. Se usa cuando el orden de los elementos sí importa, como pasos de un proceso.

Sintaxis básica

<ol>
  <li>Instalar VS Code</li>
  <li>Crear el archivo index.html</li>
  <li>Abrir en el navegador</li>
</ol>

Uso adecuado

Para instrucciones paso a paso, recetas, rankings o procedimientos donde el orden determina el resultado.

Buenas prácticas

  • Usar el atributo start="5" para comenzar desde un número diferente al 1.
  • Cambiar el estilo de numeración con CSS: list-style-type: upper-roman.

<li> — Elemento de lista

Definición

Define cada ítem individual dentro de una lista. Funciona tanto dentro de <ul> como de <ol>.

Sintaxis básica

<ul>
  <li>Primer elemento</li>
  <li>Segundo elemento</li>
  <li>Tercer elemento</li>
</ul>

Uso adecuado

Solo puede usarse como hijo directo de <ul> u <ol>. Puede contener texto, imágenes, enlaces o incluso otras listas anidadas.

Buenas prácticas

  • Puede contener cualquier contenido HTML dentro.
  • Para listas anidadas, colocar la nueva <ul> u <ol> dentro del <li>.

<dl> <dt> <dd> — Lista de definiciones

Definición

<dl> es el contenedor de la lista. <dt> es el término a definir. <dd> es la descripción o definición de ese término.

Sintaxis básica

<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado para páginas web.</dd>
  <dt>CSS</dt>
  <dd>Lenguaje para dar estilos a páginas web.</dd>
</dl>

Uso adecuado

Para glosarios, diccionarios técnicos, secciones de preguntas frecuentes (FAQ) o cualquier par término-descripción.

Buenas prácticas

  • Un <dt> puede tener varios <dd> si el término tiene varias definiciones.
  • No usar para listas comunes; solo cuando hay relación término-descripción.

📝 Etiquetas de Formularios

Permiten al usuario ingresar y enviar datos. Son fundamentales para registros, búsquedas y contacto.

<form>

Definición

Contenedor del formulario. Define hacia dónde se envían los datos (action) y cómo se envían (method: GET o POST).

Sintaxis básica

<form action="/procesar.php" method="POST">
  <!-- campos del formulario aquí -->
</form>

Uso adecuado

GET se usa para búsquedas (los datos se ven en la URL). POST se usa para enviar datos sensibles como contraseñas o información personal.

Buenas prácticas

  • Siempre indicar el atributo method; el valor por defecto es GET.
  • Usar POST para formularios con contraseñas o datos personales.

<input>

Definición

Campo de entrada de datos. Con el atributo type se puede convertir en campo de texto, correo, contraseña, número, fecha, checkbox, radio y más.

Sintaxis básica

<input type="text"  name="nombre"   placeholder="Tu nombre" required />
<input type="email" name="correo"   placeholder="Tu correo" />
<input type="password" name="clave" placeholder="Contraseña" />
<input type="number" name="edad"   min="1" max="100" />
<input type="checkbox" name="acepto" /> Acepto los términos

Uso adecuado

Es una etiqueta vacía (no tiene cierre). El atributo name es lo que identifica el campo cuando el servidor recibe los datos.

Buenas prácticas

  • Siempre asociar con un <label> usando los atributos id y for.
  • Usar el type correcto activa el teclado adecuado en celulares (ej: type="email" muestra el teclado con @).
  • Usar required para hacer un campo obligatorio.

<label>

Definición

Etiqueta descriptiva de un campo del formulario. Mejora la accesibilidad porque al hacer clic en el label, el cursor va directamente al campo.

Sintaxis básica

<label for="correo">Correo electrónico:</label>
<input type="email" id="correo" name="correo" />

Uso adecuado

El valor del atributo for del label debe ser igual al id del input que describe. Así quedan asociados.

Buenas prácticas

  • Nunca omitir el <label>; los lectores de pantalla lo usan para describir el campo a personas con discapacidad visual.
  • No usar solo el atributo placeholder como reemplazo del label.

<textarea>

Definición

Área de texto de varias líneas. Ideal para mensajes, comentarios o descripciones largas que no caben en un <input>.

Sintaxis básica

<label for="mensaje">Mensaje:</label>
<textarea id="mensaje" name="mensaje"
          rows="4" placeholder="Escribe tu mensaje...">
</textarea>

Uso adecuado

A diferencia de <input>, sí tiene etiqueta de cierre. El número de líneas visibles se controla con el atributo rows.

Buenas prácticas

  • Usar CSS resize: vertical para que solo se pueda redimensionar hacia abajo.
  • Usar maxlength para limitar el número de caracteres permitidos.

<select> y <option>

Definición

Menú desplegable. <select> es el contenedor y <option> define cada opción que el usuario puede elegir.

Sintaxis básica

<label for="nivel">Nivel:</label>
<select id="nivel" name="nivel">
  <option value="">-- Selecciona --</option>
  <option value="basico">Principiante</option>
  <option value="intermedio">Intermedio</option>
  <option value="avanzado" selected>Avanzado</option>
</select>

Uso adecuado

Para listas largas de opciones predefinidas. El atributo value de cada option es lo que recibe el servidor, no el texto visible.

Buenas prácticas

  • Incluir siempre una opción vacía o de instrucción al inicio ("-- Selecciona --").
  • Usar el atributo selected para marcar la opción predeterminada.

<button>

Definición

Botón clicable. Con type="submit" envía el formulario. Con type="reset" limpia todos los campos. Con type="button" no hace nada por defecto (se usa con JavaScript).

Sintaxis básica

<button type="submit">Enviar formulario</button>
<button type="reset">Limpiar campos</button>

Uso adecuado

Dentro de un <form>, el botón con type="submit" envía los datos automáticamente sin necesitar JavaScript.

Buenas prácticas

  • Siempre especificar el atributo type para evitar comportamientos inesperados.
  • El texto del botón debe indicar claramente qué hace: "Enviar", "Registrarse", "Buscar".

📊 Etiquetas de Tablas

Las tablas organizan datos en filas y columnas. Son ideales para comparar información, mostrar horarios o resultados.

<table>

Definición

Es el contenedor principal de toda la tabla. Todo el contenido tabular (filas, columnas, encabezados) va dentro de él.

Sintaxis básica

<table>
  <thead>...</thead>
  <tbody>...</tbody>
  <tfoot>...</tfoot>
</table>

Uso adecuado

Solo para datos tabulares: horarios, notas, comparaciones, estadísticas. Nunca para diseñar el layout de la página.

Buenas prácticas

  • Agregar <caption> para describir el contenido de la tabla.
  • Envolver la tabla en un <div> con overflow-x: auto para que funcione en móviles.

<thead> <tbody> <tfoot>

Definición

Dividen la tabla en tres partes semánticas: <thead> para los encabezados de columna, <tbody> para los datos, <tfoot> para totales o resúmenes.

Sintaxis básica

<table>
  <thead>
    <tr><th>Nombre</th><th>Nota</th></tr>
  </thead>
  <tbody>
    <tr><td>Ana</td><td>4.5</td></tr>
  </tbody>
  <tfoot>
    <tr><td>Promedio</td><td>4.5</td></tr>
  </tfoot>
</table>

Uso adecuado

Permiten aplicar estilos diferentes a cada sección de la tabla y mejoran la semántica para buscadores y lectores de pantalla.

Buenas prácticas

  • Usar siempre las tres secciones para tablas completas.
  • Al imprimir, el navegador puede repetir el <thead> en cada página automáticamente.

<tr> <th> <td>

Definición

<tr> define una fila. <th> es una celda de encabezado (negrita y centrada por defecto). <td> es una celda de datos normal.

Sintaxis básica

<!-- Fila de encabezados -->
<tr>
  <th scope="col">Aprendiz</th>
  <th scope="col">Nota</th>
</tr>

<!-- Fila de datos -->
<tr>
  <td>Ana García</td>
  <td>4.8</td>
</tr>

Uso adecuado

Con colspan una celda puede ocupar varias columnas; con rowspan puede ocupar varias filas.

Buenas prácticas

  • Siempre usar <th> para la fila del encabezado, no <td> con negrita manual.
  • Usar el atributo scope="col" o scope="row" en <th> para mejorar accesibilidad.

<caption>

Definición

Agrega un título o descripción a la tabla. Aparece visualmente encima de la tabla y es leído por lectores de pantalla antes que el contenido.

Sintaxis básica

<table>
  <caption>Notas finales del período — Grado 10°</caption>
  <thead>...</thead>
  <tbody>...</tbody>
</table>

Uso adecuado

Debe ir inmediatamente después de <table> y antes de cualquier otra etiqueta. Es el equivalente al "alt" de las imágenes pero para tablas.

Buenas prácticas

  • Siempre incluirlo en tablas complejas para mejorar la accesibilidad.
  • Ser descriptivo: "Notas del período" es mejor que solo "Tabla 1".

🎬 Etiquetas de Multimedia

Permiten integrar imágenes, audio, video e iframes en la página sin necesidad de plugins externos.

<img>

Definición

Inserta una imagen en la página. Es una etiqueta vacía (no tiene cierre). Los atributos src (ruta de la imagen) y alt (texto alternativo) son obligatorios.

Sintaxis básica

<img src="foto.jpg"
     alt="Descripción de la imagen"
     width="300"
     height="200" />

Uso adecuado

El atributo src puede ser una ruta relativa (foto.jpg), una ruta absoluta (/imagenes/foto.jpg) o una URL completa (https://...).

Buenas prácticas

  • Siempre incluir el atributo alt con una descripción; si la imagen es decorativa, usar alt="" (vacío).
  • Especificar width y height para evitar que la página "salte" mientras carga.
  • Usar imágenes en formato WebP para menor peso.

<video>

Definición

Reproduce un archivo de video directamente en el navegador, sin necesidad de Flash ni plugins. Soporta MP4, WebM y OGG.

Sintaxis básica

<video width="640" controls>
  <source src="video.mp4" type="video/mp4" />
  <source src="video.webm" type="video/webm" />
  Tu navegador no soporta la etiqueta video.
</video>

Uso adecuado

El atributo controls muestra los controles de reproducción. Se pueden agregar varios <source> para compatibilidad con distintos navegadores.

Buenas prácticas

  • Siempre incluir al menos dos formatos (MP4 y WebM) para compatibilidad.
  • Si usas autoplay, también debes agregar muted; los navegadores modernos lo exigen.
  • Usar poster="imagen.jpg" para mostrar una imagen mientras el video carga.

<audio>

Definición

Reproduce un archivo de audio en el navegador con controles nativos. Soporta MP3, OGG y WAV.

Sintaxis básica

<audio controls>
  <source src="sonido.mp3" type="audio/mpeg" />
  <source src="sonido.ogg" type="audio/ogg" />
  Tu navegador no soporta la etiqueta audio.
</audio>

Uso adecuado

Para podcasts, música de fondo, efectos de sonido o narración de texto. El texto dentro de la etiqueta se muestra solo si el navegador no soporta <audio>.

Buenas prácticas

  • Nunca usar autoplay en audio sin que el usuario lo haya solicitado.
  • Siempre incluir el atributo controls para que el usuario pueda pausar y ajustar el volumen.

<iframe>

Definición

Incrusta otra página web o contenido externo dentro de la página actual. Se usa para YouTube, Google Maps, formularios externos y más.

Sintaxis básica

<!-- Video de YouTube -->
<iframe
  src="https://www.youtube.com/embed/ID_DEL_VIDEO"
  width="560"
  height="315"
  title="Descripción del video"
  allowfullscreen>
</iframe>

Uso adecuado

Para incrustar contenido de terceros que ya tiene su propia URL. A diferencia de <video>, carga un documento completo externo dentro de la página.

Buenas prácticas

  • Siempre incluir el atributo title descriptivo para accesibilidad.
  • No incrustar páginas de fuentes desconocidas; puede ser un riesgo de seguridad.

<figure> y <figcaption>

Definición

<figure> agrupa un elemento multimedia (imagen, video, código) con su descripción. <figcaption> es el texto de pie o leyenda de ese contenido.

Sintaxis básica

<figure>
  <img src="diagrama.png"
       alt="Diagrama de estructura HTML" />
  <figcaption>
    Fig. 1 — Estructura básica de un documento HTML5.
  </figcaption>
</figure>

Uso adecuado

Para imágenes, diagramas o fragmentos de código que van acompañados de una descripción contextual. <figcaption> puede ir antes o después del contenido dentro de <figure>.

Buenas prácticas

  • El <figcaption> da contexto adicional que el atributo alt no puede dar.
  • Usar siempre que una imagen necesite una nota o referencia explicativa.
